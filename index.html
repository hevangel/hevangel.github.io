Where It All Began

GW-BASIC — bundled with IBM PCs and MS-DOS — was the very first screen that ever blinked back at an entire generation. Descended from Altair BASIC (1975), it was an interpreter: type, RUN, obey. No compiler, no build step. It taught millions that a computer was a thing you could talk to.

▸ 02 / Specialty

The Immediate-Feedback Machine

GW-BASIC's genius was instant gratification. Line numbers were the file system, the version control, the whole mental model. GOTO, GOSUB, PEEK/POKE: raw, unfiltered control over the hardware, with built-in graphics and sound. A teenager alone in a bedroom could make a game and watch it work in seconds.

▸ 03 / My Take

The First Spark

GW-BASIC was my "hello, world" — the moment a glowing green screen became a canvas. Today I design chips where a signal flip takes weeks to verify, but the instinct is the same: formulate, run, read the output, iterate. The loop times got longer, the dopamine hit of "it works!" was forged right here.

The Data Awakens

dBASE was the killer app of the pre-Windows business world. Born as Vulcan (1978), it turned the IBM PC into a database machine. Its terse dot-prompt language shipped real accounting and inventory systems. It quietly invented the 4GL — describe what you want the data to do, not how.

▸ 02 / Specialty

Persistent Records & the Dot Prompt

A .DBF file was a real table on disk — fields, types, records. You could index it and query it relationally decades before SQL was common on PCs. The xBase dialect (FoxPro, Clipper) outlived Ashton-Tate itself.

▸ 03 / My Take

Learning That Data Has Shape

dBASE taught me that programming isn't only about logic — it's about information. A record has a schema, a lifecycle; you index for speed. Today, when I debug cache-coherency bugs that span firmware, OS, and silicon, every one is at heart a data integrity problem. I started asking "what is canonical?" at an amber dot prompt.

The Pipe & the Filter

The Unix shell — sh, csh, and especially tcsh on Berkeley Unix — is the original "glue language." Chain tiny tools with |: grep, sed, awk, sort. Composability as a philosophy. tcsh added filename completion, history editing, and job control.

▸ 02 / Specialty

Composable Automation

The shell's superpower is orthogonal composition: every tool reads stdin and writes stdout, so any pipeline just works. It is the duct tape that has held engineering together for fifty years.

▸ 03 / My Take

Automation as a Reflex

The shell is where I learned that if you do it twice, script it. Every chip project I run is stitched together by tcsh and Make — nightly regressions, coverage merges, log scraping. The AI chips I help build are born inside shell scripts that kick off a thousand simulations and report back in green PASS.

Embeddable Glue

Tcl — the Tool Command Language — was created by John Ousterhout at Berkeley as a tiny, embeddable scripting engine. Pair it with Tk, its GUI toolkit, and you could build a cross-platform windowed app in a dozen lines. EDA vendors loved it — the command consoles inside virtually every chip-design tool speak Tcl.

▸ 02 / Specialty

The Language Inside the Tools

Tcl's specialty is being hosted. "Everything is a command, everything is a string." Minimalist enough to drive a tool, flexible enough to script complex flows. Tk proved GUIs didn't require C++ and a week of boilerplate.

▸ 03 / My Take

Driving the EDA Engines

Tcl is the common tongue of chip design. Whether I'm scripting synthesis or automating place-and-route, the console is almost always Tcl. It taught me that a powerful tool is really a programmable tool — the best engineers don't click buttons, they script the engine. Every tapeout I've shipped has Tcl threading through its spine.

C with Classes

C++ began in 1979 as "C with Classes" and grew into the heavyweight champion of systems programming. It gave C a layer of object orientation, templates, and RAII. Browsers, databases, game engines, OSes — and crucially, EDA tools themselves — are written in C++. It is the bedrock language of performance.

▸ 02 / Specialty

Zero-Cost Abstraction

C++ promises towering abstractions that compile down to code as tight as hand-written C. "You don't pay for what you don't use." It's the language for when performance is non-negotiable.

▸ 03 / My Take

Building the Tools That Build Chips

The simulators and synthesizers I use every day — the tools that model a billion-transistor chip — are themselves C++ programs. Writing C++ taught me to design fast, layered systems, which is exactly the mindset RTL demands. When I optimize a datapath for timing, I'm thinking in the same cache-lines-and-throughput mental model C++ beat into me.

Write Once, Run Anywhere

Java landed in 1995 like a meteor. Repackaged for the Web with mascot Duke and a steaming coffee cup, its pitch was almost religious: compile to a virtual machine and run on any device. Applets put code inside the browser; then Java quietly conquered the server, where it rules enterprise back-ends to this day.

▸ 02 / Specialty

The JVM & Garbage Collection

Java's gifts were the JVM — abstracting the OS, pioneering JIT — and automatic garbage collection as a default. Spring, Hadoop, Kafka, Android all run on its descendants.

▸ 03 / My Take

Abstraction as Leverage

Java taught me that abstraction is leverage, not laziness. GC freed my brain to think about behavior, not memory — exactly the headroom I need to reason about a billion-transistor chip. But Java also taught me the cost of abstraction: every layer hides latency. In silicon, those hidden costs are nanoseconds and watts.

Brendan Eich's Ten Days

JavaScript was written in ten days in 1995 — originally Mocha, renamed to ride Java's coattails, with nothing to do with Java. A joke for a decade, then Ajax and Gmail proved the browser was a real platform. With Node.js it escaped the browser and became the most widely deployed runtime in history.

▸ 02 / Specialty

The Event Loop & Universal Reach

JS's superpower is its single-threaded event loop — a model built for waiting. Its other superpower is ubiquity: every device has a JS engine. One language, front to back.

▸ 03 / My Take

Thinking in Events, End to End

JS rewired my brain to think in events and reactions: a promise resolves, a clock edge arrives, a signal settles — the next stage reacts. It also made me a full-stack generalist. Holding the speed of the web and the rigor of silicon in one head keeps me humble — and this very page is hand-written JS.

Optimized for Joy

Yukihiro "Matz" Matsumoto designed Ruby with one goal: programmer happiness. Everything is an object. Blocks, closures, and a syntax that reads almost like English. When Rails arrived in 2004, Ruby became the fastest way to ship a web app, and a generation of startups was born on it.

▸ 02 / Specialty

Expressiveness & DSLs

Ruby's specialty is malleability — you can bend its syntax into a domain-specific language. Rails reads like declarative poetry: has_many :comments. It prioritized developer flow over machine speed, a radical idea at the time.

▸ 03 / My Take

The Value of Beauty

Ruby taught me that code can be beautiful, and beauty matters. After years of terse C and rigid Java, Ruby's expressiveness felt like oxygen. It trained me to value readability and developer experience — principles I now apply to RTL testbenches and build scripts. Elegance is a form of engineering discipline, not vanity.

Batteries Included

Python, named after Monty Python, was built to be readable above all. NumPy, Pandas, and PyTorch made it the lingua franca of AI. Today, every neural network training run on Earth begins with import torch.

▸ 02 / Specialty

The Universal Glue

Python's specialty is breadth: it wraps C/C++/Fortran libraries, scripts EDA flows, runs web servers, and trains trillion-parameter models. Readable enough for a beginner, powerful enough to run the world's AI.

▸ 03 / My Take

The Bridge to AI Silicon

Python is the bridge between my two worlds. The AI models that my chips accelerate are defined and trained in Python; the scripts that verify those same chips are Python too. The engineer who speaks both PyTorch and RTL is the one who can actually ship the silicon that powers it. Python closed that loop for me.

The Machine's Native Tongue

Assembly is the human-readable shadow of the CPU's actual instruction set. Each maps almost one-to-one onto the binary opcodes the silicon executes. Before compilers were trustworthy, it was how everything got written — OSes, games, BIOS routines.

▸ 02 / Specialty

Total Control, Zero Safety Net

Every register, every flag, every memory address — no GC, no type system, no bounds check. Its specialty is transparency: reading disassembly is how you learn what a compiler really did. The basement beneath every other language here.

▸ 03 / My Take

Where Software Meets Silicon

Assembly was the bridge that turned me from a programmer into an engineer. The day 0x7C00 lit up in a debugger, the abstraction curtain tore. Today, when a branch predictor mispredicts, the debug trail ends in instruction-level behavior — and I can draw a straight line from the RTL waveform to the disassembly. It's the Rosetta Stone between my day job and everything else.

Describing Silicon Before It Exists

VHDL — IEEE 1076, 1987 — is one of two languages used to design every digital chip on Earth. The mind-flip: it's not a program that runs — it's a description of a circuit that exists. Write c <= a and b; and you've specified a physical AND gate etched into silicon.

▸ 02 / Specialty

Concurrency, Signals & Simulation

Everything in VHDL happens in parallel — because that's how hardware works. You simulate before fabrication because you can't patch silicon. A bug costs millions and months.

▸ 03 / My Take

My Day Job, My Native Layer

VHDL is where my journey converges. Everything above is built on hardware described in languages like this. When a bug spans firmware, OS, and silicon, I chase it from RTL waveform up to the JS console and back. Fifty years in, the loop is closed: the kid who typed GOTO 10 now designs the gates that fetch it.

One Language, Two Jobs

SystemVerilog (IEEE 1800) merged Verilog's design syntax with a powerful OOP verification layer. Then UVM made it possible to build reusable, randomized test environments for the most complex chips ever built. It is the industry standard for taping out modern silicon.

▸ 02 / Specialty

Constrained-Random Verification

SystemVerilog's superpower is constrained random testing: describe the legal input space, and the solver blasts millions of edge-case scenarios at your design, hunting bugs no human would think to write. Combined with functional coverage, it turns "did we test enough?" into a measurable number.

▸ 03 / My Take

Proving the Chip Before It's Real

SystemVerilog + UVM is how I sleep at night before a tapeout. On an AI accelerator, you can't test every MAC by hand — you let the randomized environment find the one-in-a-billion collision. Verification is harder than design, and coverage is a moral contract with the silicon.

The Pioneer of Random Verification

The e language, created by Yoav Hollander at Verisity (later Cadence), was the original constrained-random verification language — predating SystemVerilog/UVM by years. Its Aspect-Oriented features let engineers inject checks anywhere without touching the original code.

▸ 02 / Specialty

Aspects & Temporal Checking

e's specialties are Aspect-Oriented Programming — weave new behavior into existing code without modifying it — and powerful temporal assertions that check sequences over time. It made verification a first-class engineering discipline.

▸ 03 / My Take

Learning to Trust Randomness

e taught me a counterintuitive truth: the best tester is one that surprises you. Hand-written tests find the bugs you imagined; random verification finds the ones you didn't. That philosophy — define the rules, then let the machine explore — is exactly how I think about AI today.
